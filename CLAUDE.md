This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Workers' compensation IME (Independent Medical Examination) scheduling portal for healthcare support staff. Staff book patients with doctors at specific locations and time slots. The system tracks appointments through a 13-state lifecycle (Pending ‚Üí Approved ‚Üí CheckedIn ‚Üí Billed, etc.), manages doctor availability slots, and links applicant attorneys to appointments. Multi-tenant SaaS: transactional data (Appointments, DoctorAvailabilities) is tenant-scoped; reference data (States, AppointmentTypes, AppointmentLanguages) is host-scoped with per-tenant hide/show on top (intent per `docs/product/`, not yet in code); Locations are tenant-specific by intent (currently host-scoped in code -- tenant-scoping is a follow-up build item); the AppointmentStatus entity is being dropped at MVP per `docs/product/appointment-statuses.md`.

## Critical Constraints

**Windows: use a short project path** ‚Äî if the project path exceeds ~200 chars, `Microsoft.Data.SqlClient.SNI.dll` fails to load (260-char native DLL limit). Clone to a short path or use `subst` to create a drive alias. Not an issue on macOS/Linux.

**Never `ng serve` or `yarn start`** ‚Äî Angular 20's Vite pre-bundler splits `@abp/ng.core` across chunks, creating duplicate `InjectionToken` instances. Angular DI uses `===` identity, so `CORE_OPTIONS` injection fails at runtime. Use `ng build --configuration development` + `npx serve` instead.

**Start services in order: AuthServer ‚Üí HttpApi.Host ‚Üí Angular** ‚Äî the API host validates tokens against AuthServer; Angular calls both.

**Start SQL Server before dotnet services** ‚Äî cold start causes connection errors during ABP's permission seeding. Start your SQL Server instance (Docker, LocalDB, or full) before running any .NET service.

**Never edit files in `angular/src/app/proxy/`** ‚Äî these are auto-generated by ABP's proxy generator. Run `abp generate-proxy` to regenerate after backend changes.

## Build & Run Commands

```bash
# 1. Restore .NET packages
dotnet restore

# 2. Start SQL Server (Docker, LocalDB, or full ‚Äî must be running before .NET services)

# 3. Run AuthServer ‚Äî wait for 200 on /.well-known/openid-configuration (Terminal 1)
dotnet run --project src/HealthcareSupport.CaseEvaluation.AuthServer

# 4. Run API Host ‚Äî wait for Swagger 200 (Terminal 2)
dotnet run --project src/HealthcareSupport.CaseEvaluation.HttpApi.Host

# 5. Build and serve Angular (Terminal 3) ‚Äî never use ng serve or yarn start
cd angular
npx ng build --configuration development
npx serve -s dist/CaseEvaluation/browser -p 4200

# Verify all three services
curl -sk -o /dev/null -w "%{http_code}" https://localhost:44368/.well-known/openid-configuration  # 200
curl -sk -o /dev/null -w "%{http_code}" https://localhost:44327/swagger/index.html                # 200
curl -s  -o /dev/null -w "%{http_code}" http://localhost:4200/                                    # 200
```

Two DbContext classes exist: `CaseEvaluationDbContext` (host + tenant, `MultiTenancySides.Both`) and `CaseEvaluationTenantDbContext` (tenant-only). Host-only entity configs are guarded with `if (builder.IsHostDatabase())`.

**Test base class chain:** `CaseEvaluationTestBase<TModule>` (Autofac, `appsettings.json` loaded) ‚Üí `CaseEvaluationApplicationTestBase` or `CaseEvaluationDomainTestBase` ‚Üí concrete test class. EF Core tests use SQLite in-memory (`AbpEntityFrameworkCoreSqliteModule`) and require `[Collection(CaseEvaluationTestConsts.CollectionDefinitionName)]`.

**Test data seeding:** `DoctorsDataSeedContributor` seeds 2 doctors with hardcoded GUIDs ‚Äî tests assert against those specific IDs. Add new seed contributors for new entity tests.

**Coverage gap:** Only Doctors, Books, and framework services have tests. Patients, Appointments, Locations, DoctorAvailabilities, and all others are untested.

## ABP Conventions

**DTO naming:** `{Entity}CreateDto`, `{Entity}UpdateDto`, `{Entity}Dto`, `{Entity}WithNavigationPropertiesDto`, `Get{Entities}Input`. Do NOT use `CreateUpdate{Entity}Dto`.

**AppService:** Extend `CaseEvaluationAppService` (not `ApplicationService` directly). Always add `[RemoteService(IsEnabled = false)]` ‚Äî omitting it causes ABP to expose duplicate routes via both the controller and the service.

**Mappers:** Add a `partial class` decorated with `[Mapper(RequiredMappingStrategy = RequiredMappingStrategy.Target)]` extending `MapperBase<TSource, TDest>` in `CaseEvaluationApplicationMappers.cs`. Riok.Mapperly generates the implementation at compile time.

**Permissions:** Add a nested static class to `CaseEvaluationPermissions.cs` AND register child permissions in `CaseEvaluationPermissionDefinitionProvider.cs`. Pattern: parent permission = `Default`, children = `Create/Edit/Delete`. Dashboard permissions use `MultiTenancySides.Host` / `.Tenant`.

**Controllers:** NOT auto-wired. Every AppService needs a manual controller in `src/.../HttpApi/Controllers/` that extends `AbpController, I{Entity}AppService` and delegates each method. Route: `[Route("api/app/{entity-plural}")]`.

**Localization:** JSON files in `src/.../Domain.Shared/Localization/CaseEvaluation/`. Use `L("Key")` in permission providers; use `| abpLocalization` pipe in Angular templates.

## Multi-tenancy Rules

**7 entities implement `IMultiTenant`:** Doctor, Appointment, DoctorAvailability, ApplicantAttorney, AppointmentAccessor, AppointmentApplicantAttorney, AppointmentEmployerDetail. Tenant filtering is automatic via ABP's data filter ‚Äî no manual `WHERE TenantId = X` needed. **Note:** Patient has a `TenantId` property but does NOT implement `IMultiTenant` ‚Äî ABP's automatic tenant filter does not apply to Patient.

**Reference entities are host-only** (no `IMultiTenant`): State, WcabOffice, AppointmentType, AppointmentLanguage. Configuring these in `OnModelCreating` requires the `if (builder.IsHostDatabase())` guard. **Location** is currently host-scoped in code BUT intent (`docs/product/locations.md`) is tenant-specific; the tenant-scoping refactor is a follow-up build item -- after that ships, `Location` joins the IMultiTenant set above. The **AppointmentStatus entity is being dropped at MVP** per `docs/product/appointment-statuses.md` (the `AppointmentStatusType` enum + `en.json` localization replaces it); do not add new code referencing the entity.

**To access cross-tenant data from Host context:** inject `IDataFilter` and `using (_dataFilter.Disable<IMultiTenant>()) { ... }` ‚Äî see `DoctorsAppService` for the pattern. Never do this in tenant-context code.

**To switch tenant context explicitly:** `using (CurrentTenant.Change(tenantId)) { ... }` ‚Äî used in `DoctorTenantAppService` for tenant provisioning only.

## What Claude Should Never Do

- **Edit `angular/src/app/proxy/`** ‚Äî auto-generated; changes are overwritten on next `abp generate-proxy`.
- **Run `ng serve`, `yarn start`, or `ng build --watch`** ‚Äî all use Vite and cause `NullInjectorError: CORE_OPTIONS` in the browser.
- **Run dotnet from a long project path (Windows)** ‚Äî if the path exceeds ~200 chars, SNI.dll fails to load (260-char native DLL limit). Use a short path or drive alias.
- **Omit `[RemoteService(IsEnabled = false)]` on a new AppService** ‚Äî causes ABP to register duplicate routes alongside the controller.
- **Use AutoMapper or `ObjectMapper.Map<>` for new mappers** ‚Äî this project uses Riok.Mapperly; add a new `[Mapper]` partial class to `CaseEvaluationApplicationMappers.cs`.
- **Add `IMultiTenant` to host-scoped reference entities** (State, WcabOffice, AppointmentType, AppointmentLanguage) -- they are intentionally shared across all tenants (with per-tenant hide/show on top per `docs/product/` intent; the master rows still stay host-managed). **Exception:** `Location` is currently host-scoped but intent (`docs/product/locations.md`) is tenant-specific; a future build item adds `IMultiTenant` to `Location` specifically. The `AppointmentStatus` entity is being dropped at MVP -- do not add new code referencing it.
- **Modify `appsettings.secrets.json`** in any project ‚Äî contains ABP license keys. These files are not in `.gitignore`; treat as sensitive.
- **Skip the `AppointmentManager` domain service for create/update** ‚Äî business rules (slot booking, confirmation number generation `A#####`) live there, not in the AppService directly.

## Vocabulary

- **Promotion cascade** -- the branch-promotion flow `main -> development -> staging -> production`, where `main` is the default branch and integration trunk. The first two hops auto-open promotion PRs (`auto-pr-dev.yml` fires on push to `main`; `deploy-dev.yml` fires on push to `development`); `staging -> production` is always a manually-opened PR. One direction only; never merge a downstream branch back into an upstream branch. When Adrian says "the cascade" in conversation, he means this.
