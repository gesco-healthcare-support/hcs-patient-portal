## Recommended approach

- **Target: Azure** (HIGH for direction, MEDIUM on exact service). Single-dev operator, .NET 10, LeptonX, existing `deploy-dev.yml`/`promote-staging.yml`, first-party ABP Azure guides. AWS is viable but every tutorial must be self-mapped.
- **Compute tier: Azure Container Apps (ACA)** for the monolith -- cheaper than AKS, scale-to-zero possible for dev, matches existing Docker Compose. Keep `etc/helm/` for a future consolidated AKS cluster covering MRR AI and Case Tracking too.
- **HIPAA-eligible shortlist (Azure)**: App Service or Container Apps (compute), Azure SQL Database (Business Critical or General Purpose), Azure Cache for Redis, Key Vault, Monitor + Log Analytics, Entra ID. All covered by Microsoft Product Terms BAA via VL.
- **Secrets: Azure Key Vault + Managed Identity**. `builder.Configuration.AddAzureKeyVault(uri, new DefaultAzureCredential())` gated by `!IsDevelopment()`. GitHub Actions authenticates via federated OIDC, not stored client secrets.
- **Env-config migration**: (1) replace `environment.prod.ts` localhost URLs with real values and verify `fileReplacements` fires on `ng build --configuration production`; (2) add `appsettings.Staging.json` / `appsettings.Production.json` with Key Vault references only; (3) move the live ABP NuGet key out of `NuGet.Config` (see Q-11) before any deploy.

## Gotchas / blockers

- `environment.prod.ts` still points at localhost -- any prod build today ships broken URLs. Blocker.
- ABP requires Redis for multi-instance; single-instance App Service with Always On is simpler for Phase 1.
- OpenIddict signing keys must not live on ephemeral container FS; persist to Key Vault or Blob-backed data-protection keys. Missing on current scaffold.
- Azure BAA auto-executes under Volume Licensing / EA only -- pay-as-you-go subscriptions need `https://aka.ms/BAA` self-service enrollment.
- `etc/helm/` is scaffold-only; values-azure.yaml, ingress TLS, secret refs are not production-ready.
