# P10: Undisclosed Security Vulnerabilities or Prior Incidents -- Research

**Risk tier**: Legal + Security (inheriting a compromised system)
**Status**: Open (verified 2026-04-17)
**Nature**: Partly discoverable through public databases + repo scans; full answer requires prior-developer interview.

---

- **Active scanning in the current codebase**: gitleaks, TruffleHog, CodeQL, dependency-review, OpenSSF Scorecard all wired in `.github/workflows/`.
- **29 documented issues** in `docs/issues/` (SEC-01 through SEC-05 catalogue known compliance gaps openly).

## Industry / regulatory context

- [eCFR -- 45 CFR Part 164 Subpart D (§§ 164.400-414)](https://www.ecfr.gov/current/title-45/subtitle-A/subchapter-C/part-164) -- Breach-notification obligations; 60-day ceiling.
- [HHS OCR -- Breach Portal ("Wall of Shame")](https://ocrportal.hhs.gov/ocr/breach/breach_report.jsf) -- Public searchable database. Any 500+ breach against Gesco or a predecessor product would appear here.
- [HHS -- Breach Notification Rule](https://www.hhs.gov/hipaa/for-professionals/breach-notification/index.html) -- 60-day ceiling for covered entities; BA must notify CE without unreasonable delay and no later than 60 days from discovery.
- [HHS -- HITECH Breach Notification Rule](https://www.hhs.gov/hipaa/for-professionals/breach-notification/laws-regulations/final-rule-update/hitech/index.html) -- Statutory authority.
- [California AG -- Search Data Security Breaches](https://oag.ca.gov/privacy/databreach/list) -- California's public breach database for >500-CA-resident incidents.
- [California AG -- Data Security Breach Reporting](https://oag.ca.gov/privacy/databreach/reporting) -- Filing portal; SB 446 imposes 15 days to AG.
- [CISA -- Stakeholder-Specific Vulnerability Categorization (SSVC)](https://www.cisa.gov/stakeholder-specific-vulnerability-categorization-ssvc) -- Act / Attend / Track / Track* decision tree for prioritising findings.
- [ISO/IEC 29147:2018 -- Vulnerability Disclosure](https://www.iso.org/standard/72311.html) -- International standard for coordinated disclosure.
- [BSA -- Principles for Coordinated Vulnerability Disclosure](https://www.bsa.org/files/policy-filings/2019globalbsacoordinatedvulnerabilitydisclosure.pdf) -- Industry-association framework.
- [CERT/CC -- SSVC How-To for Coordination Triage](https://certcc.github.io/SSVC/howto/coordination_triage_decision/) -- Applying SSVC to a vendor's inbound vulnerability queue.

## Likely answer

**LOW confidence on "were vulnerabilities found and suppressed"** -- this is unknowable from code alone.

What IS knowable:
- No deployed environments means no production breach history to hide (HIGH).
- Automated scanning is active, so CURRENT vulnerability posture is being tracked openly (HIGH).
- 29 catalogued issues suggest transparent handling of current-state defects (HIGH).
- Squashed-import history means any pre-import vulnerability history was silently patched or silently retained. The committed NuGet key itself is one such carry-over issue surfaced after import.

**Assumptions that would flip this:**
- The prior developer's original repo still exists with full history showing security-related commits.
- Any record of a past incident in email, helpdesk tickets, or Gesco business records.
- A hit against "Gesco" or any predecessor product name in the HHS OCR Breach Portal or the California AG list.

## Recommended approach to obtain a definitive answer

- Search [HHS OCR Breach Portal](https://ocrportal.hhs.gov/ocr/breach/breach_report.jsf) for "Gesco" and any predecessor/affiliate names.
- Search [California AG breach list](https://oag.ca.gov/privacy/databreach/list) for the same.
- Ask prior developer: "Are there any known security vulnerabilities or past incidents we should be aware of? Any CVEs ever filed? Any responsible-disclosure reports received?"
- Ask Gesco business owner: "Have we ever received a breach notification, a vulnerability report from a security researcher, or a subpoena/inquiry from HHS OCR or the California AG?"
- Run `trufflehog git file://.` with `--log-opts="--all"`; also scan dangling refs to detect secrets that may have been "fixed" without rotation.
- Before first deploy: commission a third-party pen test + SAST/DAST scan; establish a public CVD policy per ISO/IEC 29147; triage findings using CISA SSVC.

## Default assumption if unanswered

Assume no publicly reported incidents (verify via the two portals). Assume unknown number of latent vulnerabilities from the pre-import era. Treat the 2026-04-08 import as security "day zero" -- any finding is a new finding, not a suppressed one. Pre-deploy requirements: clean pen test, established CVD policy, documented SSVC triage.

## Risk if assumption is wrong

**If a past breach went unreported:** Gesco inherits the notification obligation upon discovery under § 164.400-414 (60 days) and Civ. Code § 1798.82 (now 30 days to individuals / 15 to AG under SB 446). Wilful failure to notify is a separate HIPAA violation tier. Documenting the date Gesco first learned of each issue matters -- clock runs from discovery.

**If a vulnerability was known and silently patched:** possible breach-of-duty/negligence exposure to any user whose data was exposed. Rotate keys regardless; key revocation beats history rewriting.
