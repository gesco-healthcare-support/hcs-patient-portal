# Q11: ABP Commercial License Ownership & Transfer -- Research

**Severity**: Critical (license ownership unclear; live NuGet key committed)
**Status**: Open (verified 2026-04-17)

---

## Current state (verified 2026-04-17)

- `NuGet.Config` contains a live per-user ABP API key directly in version control.
- `NuGet.Config.template` uses an env-var placeholder -- the template is clean, the checked-in config is not.
- `@volo/abp.ng.*` Angular packages require authenticated access to `npm.abp.io`.
- No license file in the repo; `LICENSE` only covers the proprietary Gesco copyright.
- No documentation of which ABP account owns the license.

## Official documentation

- [ABP Pricing](https://abp.io/pricing) -- Licenses sold per-organisation with N developer seats (Team 5, Business 10, Enterprise larger); perpetual licence, annual renewal for updates/support.
- [ABP Commercial FAQ](https://commercial.abp.io/faq) -- "Non-personal paid license ... purchased by the company and can be used by anyone within the organization." Seats reassignable without additional cost when a developer leaves.
- [ABP EULA](https://abp.io/eula) -- "Customer may not assign or otherwise transfer the Agreement ... without the written consent of Volosoft" -- account transfer is a manual Volosoft process, not self-service.
- [ABP general FAQ](https://abp.io/faq) -- Renewal/transfer contact is `marketing@abp.io`; organisation management at `https://abp.io/my-organizations`.
- [ABP License Server](https://commercial.abp.io/) -- Central licence-validation endpoint; issues and validates the NuGet API key per organisation.
- [ABP -- Licensing change notes (July 2024)](https://gist.github.com/ebicoglu/031e5d9adb4c8b6ff6aeb529290a6efe) -- Personal / Team / Business / Enterprise tier split documented.

## Community findings

- [Reddit -- ABP developer seats question](https://www.reddit.com/r/dotnet/comments/1hdjict/abp_framework_developer_seats_question/) -- Confirms org admin reassigns a departed dev's seat on `my-organizations`; no cost.
- [ABP support #5728 -- Update licence Business ABP](https://abp.io/support/questions/5728/Update-License-Bussiness-ABP) -- Volosoft handles licence-holder changes via email to `license@abp.io` / `marketing@abp.io`.
- [ABP support #2206 -- How to get API token](https://abp.io/support/questions/2206/How-to-get-API-token) -- Confirms format `nuget.abp.io/<guid>/v3/index.json`, regenerable from account page.
- [GitGuardian -- Remediating NuGet API key leaks](https://www.gitguardian.com/remediation/nuget-api-key) -- Rotation is mandatory once leaked; removing from current commit is insufficient since history preserves the key.
- [Truffle Security -- Deleting leaked keys is not a solution](https://trufflesecurity.com/blog/remediate-leaked-api-keys-with-key-rotation) -- Rotation beats history rewriting; focus on rotation + monitoring for reuse.
- [ABP support #1791 -- Commercial licence in dev mode](https://abp.io/support/questions/1791/About-the-commercial-license-in-development-mode) -- Perpetual licence still compiles/runs after lapse; loses access to new NuGet/npm versions.

## Recommended approach

- **Immediate (today, HIGH)**: Run `abp login-info` on Adrian's dev machine. Log in to `https://abp.io/my-organizations`; the Developers tab lists every seat with its email. Screenshot the org name, owner email, seat roster. Store in the Decision Log. If the listed owner is Adrian's personal email (or worse, the prior developer's), the licence is not yet Gesco-owned.
- **Short-term (this week, HIGH)**: Treat `afccfa61-0b0f-410d-ad9a-f062655770f0` as compromised -- it is in git history. Log in to `https://abp.io/account` -> NuGet API key -> Regenerate. Old key invalidates immediately. Rotate the `@volo` npm token at the same page. Replace both in GitHub Actions secrets (`ABP_NUGET_API_KEY`, `ABP_NPM_TOKEN`); ensure CI consumes the template + secret, not the checked-in file. Remove the live `NuGet.Config` from the repo; add to `.gitignore` if absent.
- **Short-term (HIGH)**: Email `license@abp.io` with current org details and request transfer of ownership to a Gesco-owned email (e.g., `dev@gesco.com` group mailbox) if not already there. Per EULA, this requires written consent; provide Gesco business details.
- **Long-term (MEDIUM)**: Add an ADR recording the ABP org owner, seat allocation, rotation procedure, break-glass owner. Gate all CI/CD on Actions secrets, not checked-in configs. Add a gitleaks rule blocking `nuget.abp.io/[a-f0-9-]{36}/`.

## Gotchas / blockers

- Live API key exists in git history; rotation (not history rewrite) is the correct remediation per GitGuardian / Truffle.
- Personal Licence is explicitly single-developer; cannot be upgraded to Team by adding seats. If current licence is Personal, repurchase at Team tier before hiring.
- If the current org owner is the prior contractor's personal email, they technically control the seat roster until Volosoft transfers. They could revoke Adrian's seat.
- `@volo` npm packages validate against the same licence server -- expired/revoked licence breaks `yarn install` in CI. Lockfiles help short-term but miss security patches.
- Perpetual vs renewal: code keeps working after lapse, but no new module downloads or upgrades.
