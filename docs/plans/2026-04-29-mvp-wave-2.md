## Verification standards (NON-NEGOTIABLE per cap)

Every Wave 2 cap is "done" only after passing the 9-item check from the build-kickoff prompt:

1. `dotnet build` clean across the solution.
2. EF migration applies cleanly on host AND tenant DbContexts (T0 fixes the tenant path).
3. Cap-specific manual smoke per the per-cap acceptance section.
4. Cross-tenant smoke (mandatory if cap reads or writes data).
5. Browser-side LCP < 3s for any cap touching Angular bundle.
6. API response < 500ms for any cap adding or modifying an endpoint.
7. Console + log clean (Angular DevTools + `docker logs main-api-1 --tail 200`).
8. Permission test as roles that should NOT have access (expect 403 / hidden UI).
9. Demo-walk regression: full Wave 0 + Wave 1 path end-to-end.

These exist because the W1 bugfix sprint shipped 5 fixes for regressions the W1 build session missed. The pattern was "build clean, looked OK in browser -> ship -> regressions surface in real use." Wave 2 closes that gap by requiring evidence-based verification at every commit.
