# Phase B-6 Tier 2 -- Secondary Entity Test Coverage

## Goal

Add integration-test coverage for the three secondary Phase-B-6 entities -- `AppointmentAccessor`, `AppointmentApplicantAttorney`, `AppointmentEmployerDetail` -- plus the Wave-2 seed infrastructure (`SeedDoctorAvailabilitiesAsync` + `SeedAppointmentsAsync`) they depend on. ## Context

Tier 1 MERGED 2026-04-24 (PR-1E #135 closed the 6-PR set). Tier 1 covered the 5 critical-path entities (Appointments Wave 1, DoctorAvailabilities Wave 1, Patients, Locations, ApplicantAttorneys) plus shared test infrastructure. Tier 2 targets the three secondary entities documented in the B-6 Kickoff §11-12 priority matrix.

All three Tier-2 entities are `IMultiTenant` and FK-depend on `Appointment`, which is NOT currently seeded in the orchestrator. We chose **Option A -- seed-infra-first** over Option B -- inline-seed-per-test, to keep the three per-entity PRs focused on AppService + repository surface and avoid duplicating FK-setup in every test body.

Constraint reaffirmed by Adrian on 2026-04-24: test data + mock orchestrators only; documented production-code gaps (permission-enforcement misconfigurations on `AppointmentAccessorsAppService` and `AppointmentEmployerDetailsAppService`) must be ENCODED in tests via `[Fact(Skip="GAP: ...")]` -- NOT fixed in-PR. See `C:\Users\RajeevG\.claude\projects\W--patient-portal\memory\feedback_encode_gaps_in_tests.md` for the canonical encoding rule.

- **Seed order breakage:** adding the two new orchestrator methods in the wrong position crashes seed. Mitigation: T1's 2 sanity tests fail loudly. Rollback: `git revert` the orchestrator edit; the 2 new method bodies are additive-only.
- **Permission-gap encoding vs fix tension:** the documented gaps on `AppointmentAccessorsAppService` (generic `[Authorize]`) and `AppointmentEmployerDetailsAppService` (mixed enforcement) are tempting to fix. DO NOT. Decision T2-4 is hard; encode as `[Fact(Skip=...)]` + INCOMPLETE-FEATURES reference.
- **AppointmentAccessor repository-registration risk:** entity is `FullAuditedEntity`, not aggregate root. If `IRepository<AppointmentAccessor, Guid>` is not auto-registered in the test module, use `IAppointmentAccessorRepository` (custom, confirmed registered) instead -- it inherits from `IRepository<T, TKey>`.
- **FEAT-14 still not fixed:** any test that would rely on FK enforcement at DB level must be skipped with an INCOMPLETE-FEATURES reference. Do not try to inline-enable FK for a single test.
