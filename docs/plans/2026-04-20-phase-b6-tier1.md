## Approach

### Design decisions

1. **Single orchestrator, not per-entity contributors.** `CaseEvaluationIntegrationTestSeedContributor : IDataSeedContributor, ISingletonDependency` lives in `TestBase/Data/` and awaits entity-level private seed methods in strict FK order. Rejected: per-entity contributors (ABP forum #571 — execution order not guaranteed), orchestrator + per-entity-contributor hybrid (unnecessary DI dance for GUIDs).

2. **Hardcoded synthetic GUIDs + `TestData` static classes per entity.** Example: `PatientTestData.Patient1Id = Guid.Parse("...")`. Bogus generates the field values (names, emails, addresses, phones), but GUIDs are deterministic for assertion-stable tests. Bogus is seeded with a constant `Randomizer.Seed = new Random(20260420)` for reproducibility.

3. **Bogus.Healthcare for shape, HIPAA-compliant output.** Patient fields (FirstName/LastName/Email/Address/SSN-shaped) generated via Bogus. **PR-0 includes a spike step to verify PHI scanner hook does not false-positive on Bogus output**; if it does, either tune the hook or override Bogus string generators to produce hex-compatible values. The `.claude/rules/test-data.md` rule updates to permit Bogus (Safe Harbor still applies; Bogus names don't match the 18 PHI categories — they're synthetic by construction).

4. **`[Collection]` retrofit happens FIRST.** Before any new test class is created, PR-0 adds `[Collection(CaseEvaluationTestConsts.CollectionDefinitionName)]` to `EfCoreDoctorsAppServiceTests` and `DoctorRepositoryTests`. Rationale: xUnit's collection fixture requires `[Collection]` on every participating test class; missing it breaks SQLite in-memory connection sharing. The fact that existing tests pass is either coincidence (single fixture instance per run) or xUnit-default behaviour we shouldn't rely on.

5. **IdentityUser seeding in its own contributor, called first by the orchestrator.** `IdentityUsersDataSeedContributor` seeds 3 synthetic users (staff admin, attorney, patient role). Because IdentityUsers are FKs for Doctor, Patient, Appointment, ApplicantAttorney, AppointmentAccessor, they must be first. Uses `IdentityUserManager` + `IIdentityUserRepository` from ABP Identity module. Roles are seeded manually via `IIdentityRoleRepository` if needed.

6. **TDD for gap-encoding tests; code for CRUD.** Per `.claude/rules/rpe-workflow.md`:

   - `tdd`: no-state-machine, slot-not-released-on-delete, A99999 overflow, Create/Edit permission gaps, Patient non-IMultiTenant cross-tenant visibility, GeneratePreviewAsync conflict detection.
   - `code`: routine CRUD tests, nav-property queries, list-filter tests, seed contributors.

7. **One PR per entity, strict order.** PR-0 first, then 1A-1E sequentially. Each PR must: build `-warnaserror` 0/0, all existing tests pass, new tests pass, SonarCloud coverage delta is measurable (non-zero), commit convention `test({entity-kebab}): add CRUD + edge cases for {Entity} (B-6 T1)`.

### Rejected alternatives

- **Playwright / Cypress E2E in B-6 scope** — belongs in Layer 3 (Phase C+). Deferred.
- **Migrate Karma to Vitest in B-6** — Angular tests explicitly out of scope. Defer.
- **Refactor GeneratePreviewAsync inside PR-1B** — violates "test-only PRs" rule. Refactor is a separate P-11 PR after PR-1B lands.
- **Bundle Tier 1 entities into one mega-PR** — too large for review; breaks the "one entity per PR" rule.
- **One `IDataSeedContributor` per entity** — non-deterministic order. Rejected (see Decision 1).
- **Full Bogus.Healthcare (including diagnoses, ICD codes)** — not needed for Tier 1; Patient entity doesn't store diagnoses. Keep Bogus usage minimal (Name/Address/Internet modules only) for Tier 1.

## Risk / Rollback

### Risks

1. **PHI scanner hook false-positives on Bogus output** (HIGH likelihood, MEDIUM impact). Mitigation: PR-0 includes explicit spike step; tune hook or override Bogus generators to hex fallback via `TestStringUtility`. Rollback: revert PR-0's Bogus adoption; switch to hex-only per existing test-data.md rule.

2. **`[Collection]` retrofit breaks existing 13 tests** (LOW likelihood, HIGH impact). Mitigation: PR-0 runs full `dotnet test` locally before commit. Rollback: revert the 2 one-line attribute adds; tests return to current state.

3. **Orchestrator seed contributor conflicts with ABP built-in seed contributors from Identity / SaaS modules** (MEDIUM likelihood, MEDIUM impact). Mitigation: orchestrator only covers Tier-1 test entities; does not disable ABP Identity's own seed. Rollback: convert orchestrator back to one-contributor-per-entity if ABP module init breaks.

4. **TreatWarningsAsErrors=true triggers a wave of warnings across src/ that weren't caught before** (MEDIUM likelihood, MEDIUM impact). Mitigation: run `dotnet build -warnaserror` immediately after the flag flip; fix warnings as they appear OR move the flag flip to its own sub-PR. Rollback: revert Directory.Build.props diff.

5. **Bogus.Healthcare license conflict** (LOW likelihood, HIGH impact — licence risk). Mitigation: verify license before PR-0; Bogus is Apache-2.0 per GitHub; Bogus.Healthcare listed as "premium data set" — confirm free usage tier. Rollback: drop Bogus.Healthcare, use Bogus core only.

6. **SQLite in-memory can't reproduce a specific multi-tenant bug that SQL Server exposes** (LOW likelihood, LOW impact for Tier 1). Mitigation: note the SQLite-vs-SQL-Server divergences in each test class as comments; defer Testcontainers migration to Phase C.
