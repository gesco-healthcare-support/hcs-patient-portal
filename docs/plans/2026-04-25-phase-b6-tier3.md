# Phase B-6 Tier 3 -- Host-Only Lookup Entity Coverage

## Goal

Add integration-test coverage for the 5 host-only lookup entities -- `States`, `AppointmentTypes`, `AppointmentStatuses`, `AppointmentLanguages`, `WcabOffices` -- closing out Phase B-6's per-entity test work. Advance SonarCloud overall coverage from the 27.2% Tier-2 exit toward ~30-37%. Stay strictly test-data + mock-orchestrator only; no production-code edits.

## Risk / Rollback

- **Seed-extraction refactor risk (PR-3A + PR-3B):** moving State1 / AppointmentType1 inserts out of `SeedLocationsAsync` could break existing Tier-1/Tier-2 tests that assert against `LocationsTestData.State1Id`. **Mitigation:** add the new `Seed{Entity}sAsync()` method BEFORE removing the old inline insert; run `dotnet test`; only then remove the duplicate. Both Stop B steps verify this explicitly.

- **GUID prefix collision:** new prefixes `g`/`h`/`i`/`j`/`k` claimed by Tier 3. Confirmed unused by exploration agent. Stop B grep verification before each TestData merge.

- **AppointmentStatuses `DeleteAllAsync` test wipes seed state:** test #8 of PR-3C cannot wipe shared seed without breaking subsequent tests. Mitigation: skip with `[Fact(Skip="bulk-delete-against-seed-data-not-supported")]` OR redesign to never call DeleteAllAsync against shared seed. Decision at PR-3C Stop B.

- **Permission-gap tests becoming stale:** if a future production-code change adds the missing class-level Default permission to AppointmentTypesAppService, the `[Fact(Skip="GAP:...")]` test should flip live. Skip messages cite CLAUDE.md / INCOMPLETE-FEATURES anchors so future sessions find them.

- **Excel-export deferral leak:** Tier-3 must NOT add export tests. Stop B reviews PR-3E test surface for any `IRemoteStreamContent` or `GetListAsExcelFileAsync` references.

- **Branch deletions:** NEVER without Adrian's explicit OK.

- **Production-code drift temptation:** if a Tier-3 test cannot be written without a production-code change, STOP and ask Adrian at Stop B of the affected PR.
