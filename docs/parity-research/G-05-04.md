## 1. ID & Summary
The daily internal-staff "queue heads-up" digest. OLD ran
`SchedulerDomain.AppointmentApproveRejectInternalUserNotification()` (ReminderTypes id 1),
calling proc `spAppointmentApproveRejectInternalUserNotification` which RETURNED the
recipient set (`EmailList`/`PhoneList`) plus PendingAppointmentCount + ApprovedAppointmentCount.
OLD sent both an SMS (Twilio) and an email. NEW resolves recipients by ABP role
(Staff Supervisor + Clinic Staff), publishes one per-staff email event, and drops SMS for
Phase 1. Two deltas: (a) recipient targeting model, (b) SMS leg removed.

## 4. Expected vs Actual (brief)
- Expected (OLD): proc-targeted recipient list gets SMS + email with queue counts.
- Actual (NEW): Staff Supervisor + Clinic Staff role members get email-only with the same
  two counts. IT Admin / admin / Doctor excluded by design.
- Behavioral outcome for the email channel is equivalent (same two counts delivered to
  internal staff). Divergences are the channel (no SMS) and the targeting mechanism.

## 5. Repro Status
n/a-excluded. No defect to reproduce -- both deltas are deliberate design choices, confirmed

and unnecessary given static confirmation.

## 6. Why Excluded + Revisit-If
Excluded because both deltas are intentional, not regressions:
- Recipient model: OLD's targeting lived inside the SQL proc (returned `EmailList`/`PhoneList`).
  The proc body is not in the C# source, so the exact OLD recipient set cannot be reconstructed
  from code. The NEW role-based model (Staff Supervisor + Clinic Staff) is a cleaner, auditable
  equivalent and is the chosen design. No parity loss worth chasing.
- SMS leg: deliberately deferred Phase 1, tracked under the global SMS deferral
  (G-04-01 / G-05-INTENT-01), planned ~2026-08-29 when a Twilio/ACS provider lands.

Revisit IF: (1) the Twilio/ACS SMS rollout lands -- then re-add the SMS copy to this digest as
part of G-05-INTENT-01 / G-04-01 work; OR (2) someone with OLD DB access recovers the proc body
and finds the OLD recipient set materially differed from "all Staff Supervisor + Clinic Staff"
(e.g. it filtered to a per-office subset), in which case the role-based model would need
re-scoping.

## 7. Candidate Solution
None -- excluded. (If revisited for SMS: add SMS dispatch in NEW NotificationDispatcher once a
provider exists; tracked under G-04-01 / G-05-INTENT-01, not here.)
