## 1. ID & Summary
OLD has a `spm.AppointmentWorkerCompensations` table/entity that NEW does not
port. It holds `ClaimNumber`, `DateOfInjury`, `WcabAdj`, `WcabOfficeId`,
`AppointmentId` -- a strict subset of the injury/claim data already carried by
`AppointmentInjuryDetail`, which IS ported. Excluded as a dead/orphaned OLD
artifact with no unique data and no live read/write path.

## 6. Why Excluded + Revisit-If
Excluded because the entity is an orphaned/superseded parallel model of injury-
claim data, fully duplicated by `AppointmentInjuryDetail` (which is a superset).
No unique fields, no nav collection on Appointment, no read/write code path.
Revisit IF a downstream feature auditor finds OLD code that actually writes to or
reads from `spm.AppointmentWorkerCompensations` (e.g. a stored proc, a report
query, or a data-migration job referencing the table by raw SQL outside the
EF model), OR if production OLD data shows non-empty rows whose claim/injury
values diverge from the matching AppointmentInjuryDetail row.
