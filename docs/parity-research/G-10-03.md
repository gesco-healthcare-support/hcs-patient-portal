## 1. ID & Summary
OLD had a `dbo.Countries` lookup carrying per-country formatting metadata
(date/phone/postal/currency masks) plus a `City.CountryId` FK. NEW has no
Country entity, DbSet, or proxy anywhere. The live OLD app only ever used
USA, so no user-visible behavior depends on the table. Dispositioned
`defer` (not exclude) because the broader lookup-modernization item
G-07-08 is `include` and the City picker (G-10-04) proceeds; only the
Country-table half is held back until the app is genuinely multi-country.

## 4. Expected vs Actual
- Expected (OLD): a Country lookup feeds a cascading Country -> State ->
  City address picker and supplies per-country format masks.
- Actual (NEW): no Country entity. Address forms use free-text City
  (G-10-04) and State stays a name-only lookup (`States.State`, present).
  Number/date/phone formatting is handled by Angular `en-US` locale, not
  a DB row. For a single US clinic this is fully equivalent in outcome.

## 7. Candidate Solution (high-level; deep sourcing deferred)
If/when un-deferred, ride the G-07-08 structured-lookup slice:
- Add a small `Countries.Country` reference entity in Domain (Name + ISO
  code; the OLD format-mask columns are NOT worth porting -- Angular
  locale owns formatting, so they are dead metadata) seeded with USA.
- Add `CountryId` to the City lookup (G-10-04) to restore the cascading
  Country -> State -> City picker only if a second country exists.
- No external SDK/library is the crux here -- this is a pure internal
  port of a lookup table. Deep sourcing held until the upstream
  (multi-country need or the G-07-08 admin slice) actually lands.

## 8. Root-Cause Hypothesis
The NEW app was scaffolded US-single-clinic-first (Phase 1 = one doctor's
office, Falkinstein tenant), so the generic multi-country lookup was
dropped as YAGNI. OLD's format-mask columns (`DateFormat`, `PhoneFormat`,
`PostalCodeFormat`, `CurrencyFormat`) reveal an unrealized "support any
country" ambition that the live deployment never used -- confidence high
that nothing user-visible regressed.
