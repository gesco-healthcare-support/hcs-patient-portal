## 1. ID & Summary
OLD exposed a "download the blank Joint Agreement Letter" endpoint: `DocumentDownloadController.Get()`
at route `api/DocumentDownload/Download`, which calls `FileOperations.GetJointAgreementLetter()` to
pull a pre-authored DOCX template from the AWS `aws.jointagreementletter` blueprint folder and stream
it back. NEW has no equivalent endpoint/AppService. Disposition: EXCLUDE (light). Static evidence
confirms the blank-template endpoint was dead/unreachable from the OLD UI.

- docs/parity-review-log.csv:19 (the G-03-02 row; FLAG-FOR-CONFIRMATION note).

## 3. Affected Code (spot-checked)
OLD (the blank-template download = the G-03-02 subject):
- `PatientAppointment.Api/Controllers/Api/Core/DocumentDownloadController.cs:10` -- class-level
  `[Route("api/DocumentDownload")]` is COMMENTED OUT; the action `Get()` at `:21-30` keeps an active
  `[Route("api/[controller]/Download")]`, streams `GetJointAgreementLetter()` result as DOCX.

OLD Angular reference inventory (only caller of `api/DocumentDownload/Download`):
- `components/login/login.service.ts:18` -- `private fileDownload: string = 'api/DocumentDownload/Download';`
  declared but NEVER referenced anywhere in `LoginService` (only `api`, the forgot-password fields, and

Distinct, NOT this gap (a different endpoint): every UI button labelled
`downloadJointAgreementLetter()` (appointment-change-requests add/list, appointment-new-documents list,
appointment-documents search) calls `api/DocumentDownload/DownloadFile?filePath=...`
(`DocumentDownloadController.cs:33-51`), which downloads an already-uploaded file by path -- the document
re-download feature, not the blank template. Confirmed at e.g.
`appointment-change-request-add.component.ts:373-389`.

NEW: no `DocumentDownload`/`GetJointAgreementLetter` equivalent. NEW's `JointAgreement*`/`JointDeclaration*`
strings are the JDF upload feature (attorney UPLOADS a form), a different concept from this downloadable
blank template.

## 4. Expected vs Actual (brief)
- Expected if ported: a user (attorney) could click an in-app link to download a blank Joint Agreement
  Letter DOCX template.
- Actual in NEW: no such endpoint or button. Actual in OLD: the endpoint existed but had no reachable UI
  caller (only a dead `LoginService.fileDownload` field); the live buttons hit the unrelated `DownloadFile`
  path instead.

## 5. Repro Status
not-reproduced-static-confirmed. A runtime repro of "is the button reachable in OLD" would require
running the OLD SPA and exercising every screen, and confirming attorney usage requires production
telemetry not available here. Static read of the OLD Angular source is sufficient: no component binds to
`api/DocumentDownload/Download`. Read-only; no state mutated.

## 6. Why Excluded + Revisit-If
Excluded because the blank-template download is dead/unreachable in the OLD SPA (sole reference is an
uninvoked `LoginService` field; class route commented out; live buttons call a different endpoint). The
JDF upload flow (attorney uploads a completed form) is the surviving, ported concept; the downloadable
blank precursor template appears superseded.
Revisit if: Adrian confirms attorneys actually used the blank template (obtained it via a path not in the
SPA, e.g. a hardcoded link, the AuthServer/Razor side, or out-of-band), OR a future requirement needs an
in-app blank-letter download. If revisited, NEW work = host the template as an embedded resource (no AWS),
generate/serve as PDF per mission (DOCX -> PDF), expose a small download endpoint + UI link. No OLD logic
to port beyond "stream the blank template file."

## 7. Candidate Solution
None -- excluded.

## 8. Decision Rationale + confidence
Confidence HIGH that the endpoint was unreachable from the OLD SPA: the only Angular reference to
`api/DocumentDownload/Download` is a declared-but-unused private field, and the controller's class-level
route is commented out, leaving only the action route. Confidence MEDIUM->LOW on the broader claim "no
attorney ever used it," because out-of-band/non-SPA access cannot be ruled out from code alone. The CSV's
exclude disposition is consistent with the static evidence; the FLAG-FOR-CONFIRMATION is the only residual
risk and it is a usage question, not a code question.

## 9. Open Questions / Flags
- KEY FLAG (from CSV, still open): Did attorneys actually use the blank Joint Agreement Letter download in
  production? Code shows it was unreachable from the SPA, but non-SPA/out-of-band access is unverifiable
  here. If "yes," revisit (generate as PDF + download).
- Naming hazard for whoever implements: G-03-02 (blank template DOWNLOAD via `GetJointAgreementLetter`) is a
  DIFFERENT feature from the JDF UPLOAD flow and from the `DownloadFile` re-download buttons that share the
  `downloadJointAgreementLetter()` TS method name. Do not conflate them.
