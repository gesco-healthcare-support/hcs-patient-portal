## 1. ID & Summary
Concern: does rejecting an attorney's Joint Declaration Form (JDF) reopen the
appointment for rescheduling? Neither OLD nor NEW cascades a JDF rejection into
an appointment-status change or a reschedule request. NEW matches OLD's actual
(non-cascading) behavior. The "cascade into reschedule" in the audit brief
appears to be a spec expectation that was never realized in OLD code.

## 2. Docs Reviewed
- docs/parity-v2/03-documents-packets.md, section "### G-03-04" (lines 96-104),
  plus related rows G-03-B2 (OLD JDF "reject" never sets Rejected status) and
  E6/E7 in the "Equivalent" table.
- CSV note (docs/parity-review-log.csv): "rejection only sets RejectedById +
  status + sends email; NO reschedule cascade exists. NEW matches OLD."

## 3. Affected Code (spot-checked)
OLD (P:/PatientPortalOld):
- AppointmentJointDeclarationDomain.cs:172-178 -- internal-user reject branch:
  sets only `RejectedById = UserClaim.UserId`, RegisterDirty, Commit. No
  appointment lookup, no status mutation, no reschedule. (Note: it does not
  even set DocumentStatusId=Rejected here -- that is the separate G-03-B2 bug.)
- AppointmentJointDeclarationDomain.cs:255-277 -- SendDocumentEmail rejection
  branch: builds + sends the rejection email only. No status/reschedule side
  effect.

NEW (this repo):
- AppointmentDocuments/AppointmentDocumentsAppService.cs:567-596 (RejectAsync):
  sets Status=Rejected, RejectionReason, RejectedByUserId, ResponsibleUserId;
  publishes AppointmentDocumentRejectedEto. No appointment-status cascade.
- Notifications/Handlers/DocumentRejectedEmailHandler.cs (only subscriber to
  that Eto): dispatches the rejection email (and a RemainingDocs variant for
  package docs). No appointment-status mutation, no reschedule.
- The only JDF-driven appointment-status change in NEW is
  JointDeclarationAutoCancelJob (transitions to CancelledNoBill when NO JDF is
  uploaded by the cutoff) -- a cancel, not a reschedule, and unrelated to
  rejection.

## 4. Expected vs Actual (brief)
- Audit-brief expectation: JDF rejection should cascade into a reschedule
  request.
- Actual in BOTH apps: JDF rejection marks the document rejected and emails the
  uploader. No appointment reopen / reschedule. NEW == OLD.
