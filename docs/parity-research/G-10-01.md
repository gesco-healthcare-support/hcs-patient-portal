## 1. ID & Summary
G-10-01 is the STORAGE side of the internal threaded-notes feature: the OLD
`spm.Notes` table has no NEW entity / DbSet. G-09-04 is the matching BEHAVIOR
side (controller + domain + Angular modal). Both are deferred together because
the OLD feature was shipped DISABLED (its only UI launcher was commented out),
so no live OLD user ever reached it -- absence has effectively zero current
end-user impact. Defer until the owner decides whether to revive the feature
after production launch.

## 4. Expected vs Actual
- Expected (OLD ground truth): a `Notes` table holding multiple dated comments
  per appointment, with an edit-history chain via `ParentNoteId` + `EditNoteId`
  + `IsLatest` (each edit inserts a new row; old row retained; latest flagged).
- Actual (NEW): no Notes entity/table. Only `InternalUserComments` (one short
  overwrite field) -- no threading, no history, no per-author edit gate.
- Caveat: OLD shipped the feature DISABLED, so the "expected" behavior was not
  observable to live OLD users. The true parity question is whether to revive
  a never-live capability, which is a product decision (-> defer).

## 7. Candidate Solution (high-level; deep sourcing deferred)
Implement ONLY if G-09-04 is approved for revival. High-level approach (deep
external sourcing held until the upstream product decision lands):
- New ABP aggregate `Notes.Note` (Guid PK) under `Domain/Notes/`, with
  `AppointmentId` FK, `Comments`, and the threading fields. Two design options
  for edit history:
  1. Port OLD's verbatim chain (`ParentNoteId` + `EditNoteId` + `IsLatest`,
     insert-on-edit). Maximises behavioral parity; carries OLD complexity.
  2. Use ABP `FullAuditedAggregateRoot` + `AbpEntityPropertyChanges` for the
     edit history and keep only a parent/thread pointer. Simpler, modern,
     audit-native; needs confirmation it satisfies the author-only-edit and
     "show history" UI the OLD modal implied.
- Standard ABP wiring per CLAUDE.md: `INoteAppService` + impl extending
  `CaseEvaluationAppService` ([RemoteService(IsEnabled=false)]), manual
  controller, Mapperly mapper, permissions (internal-only), DbSet on the
  tenant DbContext, migration. Angular notes panel on the appointment-view page.
- Source authority for the port is the OLD code (cited in section 3); no
  external framework capability is the crux, so no external URLs are warranted
  until/unless option 2 (ABP entity-change history for UI display) is chosen.

## 8. Root-Cause Hypothesis
The Notes table was dropped from the NEW data model because its owning feature
(G-09-04) was latent/disabled in OLD and was therefore not part of the MVP
parity slice. This is a deliberate scope cut, not an oversight: the auditor
explicitly flags it as owner-decision, low priority. Confidence high -- both
sides (storage + behavior) are independently confirmed absent in NEW and
documented as a paired deferral in the CSV and INDEX.
