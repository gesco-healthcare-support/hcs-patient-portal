# AuthServer -- OpenIddict token issuer + identity Razor UI (port 44368)

The SPA never renders auth pages; it redirects here for all identity flows.
`AbpAccountPublicHttpApiModule` loads here only -- `/api/account/*` does NOT exist on HttpApi.Host.

---

## What lives here

- `CaseEvaluationAuthServerModule.cs` -- module wiring: OpenIddict, DataProtection, CORS, multi-tenancy, background-job disable.
- `Pages/Account/Login.cshtml.cs` -- anti-enumeration login override (see Conventions).
- `Pages/Account/ForgotPassword.cshtml.cs` -- calls `IExternalAccountAppService.SendPasswordResetCodeAsync` in-process.
- `Pages/Account/ResetPassword.cshtml.cs` -- calls `IExternalAccountAppService.ResetPasswordAsync` in-process; redirects to `?flash=password-updated`.
- `Pages/Account/ResendVerification.cshtml.cs` -- calls `IExternalAccountAppService.ResendEmailVerificationAsync` in-process; supports `?autosend=1` one-click resend.
- `Pages/Account/EmailConfirmation.cshtml.cs` -- calls `IdentityUserManager.ConfirmEmailAsync` directly; redirects to `?flash=email-verified` or `?flash=verification-invalid`.
- `Pages/Account/Logout.cshtml.cs` -- signs out all four Identity schemes, deletes `__tenant` + `XSRF-TOKEN` cookies, redirects SPA to `/?logout=true` on the same subdomain.
- `CaseEvaluationBrandingProvider.cs` -- sets app name/logo shown on Razor pages.

## Conventions

### Razor page overrides (filesystem precedence)
Every `Pages/Account/` file shadows the ABP Pro RCL page by path precedence -- no registration needed.
The Pro DLL is partially obfuscated, so never assume the base `IActionResult` concrete type;
match by URL substring (see `LoginModel.IsConfirmUserRedirect`).

### In-process DI instead of HTTP calls
`ForgotPassword`, `ResetPassword`, `ResendVerification` inject `IExternalAccountAppService`
directly via DI -- the Application module is listed in `[DependsOn(...)]` for exactly this reason.
Never replace these with HTTP calls to HttpApi.Host: it adds cert/client overhead and breaks
the tighter per-page rate-limit policies.

### Anti-enumeration on login
`LoginModel` collapses wrong-password, non-existent email, AND unverified-email outcomes into
one generic failure banner. The unverified-email branch additionally sets `IsUnverifiedEmailFailure=true`
(shows a Resend link) because Adrian accepted the limited side-channel that trade-off creates.
All other failure alerts are cleared before returning `Page()`.

### Flash banner contract (cross-page, query-string)
`?flash=` on `/Account/Login` is the only durable cross-redirect signal. TempData is lost
through the OpenIddict authorize redirect chain.

| Value | Emitted by | Banner |
| `password-updated` | `ResetPasswordModel` | success |
| `email-verified` | `EmailConfirmationModel` | success |
| `verification-invalid` | `EmailConfirmationModel` | warning |

### DataProtection shared via Redis -- IMPORTANT
Keys are persisted to `CaseEvaluation-Protection-Keys` in Redis with `SetApplicationName("CaseEvaluation")`.
Both AuthServer and HttpApi.Host must share this keyring; without Redis, email-confirmation
tokens generated by the API host fail validation on the AuthServer with `InvalidToken` (403).
Guard: the module checks `Redis:Configuration` before calling `PersistKeysToStackExchangeRedis`
so a missing config silently falls back to per-process ephemeral keys -- which breaks cross-host tokens.

### Tenant resolution (ADR-006 + ADR-007)
Resolver chain is `CurrentUser` then `HostAwareDomainTenantResolveContributor("{0}.localhost")` only.
`?__tenant=`, header, cookie, and route resolvers are DELIBERATELY removed (HIPAA cross-tenant risk).
`HostAwareDomainTenantResolveContributor` (not the stock `DomainTenantResolveContributor`) handles
the reserved `admin` subdomain so it maps to Host context rather than a 404.

### Background jobs disabled here
`IsJobExecutionEnabled = false`. AuthServer enqueues; HttpApi.Host executes.

### OpenIddict wildcard domains
`App:WildcardDomainsFormat` config drives the allowed redirect-URI list so parallel worktree
stacks on non-canonical ports can override without recompiling. Dev fallback is hardcoded
`http://{0}.localhost:{4200,44368,44327}`.

### Refresh-token replay window
Shortened to 2 seconds (`SetRefreshTokenReuseLeeway`) to shrink the stolen-token replay
window while still tolerating concurrent refresh retries on flaky connections.

### Production certificate
`openiddict.pfx` is conditionally embedded -- prod publish requires the file present and
`AuthServer:CertificatePassPhrase` set. Dev uses `AddDevelopmentEncryptionAndSigningCertificate`.

### Exception status-code map
Both AuthServer and HttpApi.Host call
`CaseEvaluationExceptionStatusCodeMappings.MapSharedRegistrationAndInternalUserCodes`
so registration + internal-user error codes produce the same HTTP status on both hosts.

### Localization overrides
`AbpUiResource` and `AccountResource` are injected with override JSON from
`/Localization/AbpUiOverride` and `/Localization/AccountOverride` respectively.
This is required because stock Razor pages use the base resources directly; `CaseEvaluationResource`
inheritance alone does not override them.

## Gotchas

- `EmailConfirmationModel` is `[AllowAnonymous]` and only handles `OnGetAsync`. HEAD
  requests from email scanners (Outlook Safe Links) do nothing -- by design, to prevent
  premature token consumption.
- `LogoutModel` redirects to the SPA on the SAME subdomain (`Request.Host` preserved,
  only port swapped to 4200). If `App:AngularUrl` port differs from 4200, the fallback
  parses it from the config URI.
- External auth providers (Google/MS/Twitter) are removed but NuGet references remain.
  Do not re-enable without a per-tenant configuration story. See audit finding D-13.
- `ReplaceEmbeddedByPhysical` for localization JSON is guarded by `Directory.Exists`;
  in Docker the source tree is not mounted, so the guard prevents an empty replacement
  that would nuke all localization keys at runtime.
- `ConfirmUser` redirect detection matches by URL substring because the Pro DLL is obfuscated
  and may return any of `RedirectToPageResult`, `RedirectResult`, or `LocalRedirectResult`.
