## Architecture Type

Multi-tenant monolith. Angular SPA bundled inside .NET project. One doctor per tenant model.

- **Frontend:** Angular 20 standalone components (no NgModules)
- **Backend:** ASP.NET Core with ABP Commercial module system
- **Database:** Single SQL Server with EF Core dual DbContext (host + tenant)
- **Auth:** OpenIddict OAuth 2.0 with Authorization Code + PKCE flow

## Critical Constraints

1. **Always run dotnet from `P:\` drive** — real path (~137 chars) exceeds 260-char native loader limit for `Microsoft.Data.SqlClient.SNI.dll`. Use `subst P: "C:\Users\RajeevG\Documents\Projects"`.

2. **Never `ng serve` or `yarn start`** — Angular 20's Vite pre-bundler splits `@abp/ng.core` across chunks, creating duplicate `InjectionToken` instances. Use `ng build --configuration development` + `npx serve` instead.

3. **Start services in order: AuthServer (44368) -> HttpApi.Host (44327) -> Angular (4200)** — API host validates tokens against AuthServer; Angular calls both.

4. **Start LocalDB before dotnet services** — cold start causes Named Pipes errors: `sqllocaldb start MSSQLLocalDB`.

5. **Never edit `angular/src/app/proxy/`** — auto-generated by ABP proxy generator. Run `abp generate-proxy` after backend changes.

6. **Never modify `appsettings.secrets.json`** — contains ABP license keys.

## Build & Run

```bash
# 0. Ensure P: drive
ls "P:\Patient Appointment Portal\hcs-case-evaluation-portal\src"

# 1. Restore
dotnet restore

# 2. LocalDB
sqllocaldb start MSSQLLocalDB

# 3. AuthServer (Terminal 1)
dotnet run --project "P:\Patient Appointment Portal\hcs-case-evaluation-portal\src\HealthcareSupport.CaseEvaluation.AuthServer"

# 4. API Host (Terminal 2)
dotnet run --project "P:\Patient Appointment Portal\hcs-case-evaluation-portal\src\HealthcareSupport.CaseEvaluation.HttpApi.Host"

# 5. Angular (Terminal 3)
cd "P:\Patient Appointment Portal\hcs-case-evaluation-portal\angular"
npx ng build --configuration development
npx serve -s dist/CaseEvaluation/browser -p 4200
```
