# Test Patterns — Patient Portal

| Component | Technology |
|---|---|
| Test framework | xUnit |
| Assertion library | Shouldly |
| DI container | Autofac (via ABP) |
| Test DB | SQLite in-memory |

## EF Core Test Collection

```csharp
[Collection(CaseEvaluationTestConsts.CollectionDefinitionName)]

{
    // Inherits all test methods from DoctorsAppServiceTests<T>
}
```

## Data Seeding

Test data is seeded via `IDataSeedContributor` implementations in `Domain.Tests/`:

| Seed Contributor | What It Creates |
|---|---|
| `DoctorsDataSeedContributor` | 2 doctors with hardcoded GUIDs (`63b171d1-...`, `b6d53903-...`) |

Tests assert against these specific GUIDs. New features need their own seed contributor.

Pattern:
```csharp
public class {Entity}DataSeedContributor : IDataSeedContributor, ISingletonDependency
{
    private bool IsSeeded = false;
    // ... inject repository + UoW manager
    public async Task SeedAsync(DataSeedContext context)
    {
        if (IsSeeded) return;
        await _repository.InsertAsync(new Entity(id: Guid.Parse("..."), ...));
        await _unitOfWorkManager!.Current!.SaveChangesAsync();
        IsSeeded = true;
    }
}
```

## Test Method Pattern

```csharp
[Fact]
public async Task GetListAsync()
{
    // Act
    var result = await _appService.GetListAsync(new Get{Entities}Input());
    // Assert
    result.TotalCount.ShouldBe(2);
    result.Items.Count.ShouldBe(2);
    result.Items.Any(x => x.Id == Guid.Parse("...")).ShouldBe(true);
}
```

Standard CRUD test methods: `GetListAsync`, `GetAsync`, `CreateAsync`, `UpdateAsync`, `DeleteAsync`.

## HIPAA: Test Data Requirements

ALL test data must be synthetic. Never use data that resembles real patients.

### Approved Synthetic Patterns

This project's existing tests use **random hex strings** for names and emails (e.g.,
`firstName: "551551e068be423cb150..."`, `email: "7c7fa4aa54e94b09adf79@07d1fd7ead804f..."`).
This is the preferred pattern — continue using it for consistency.

| Field Type | Approved Pattern | Example |
|---|---|---|
| Names | Random hex strings (50 chars) | `"c014822702a54810a377d172f55e915329a52881e14c4dbb90"` |
| Email | Random hex @ random hex `.com` | `"27ff91b42eed448e91265@e00ce97ffe31409791156.com"` |
| Phone | 555 prefix reserved range | `"(555) 555-0100"` |
| Address | Obviously fake | `"123 Test Street, Anytown, TS 00000"` |
| Date of birth | Obviously fake dates | `DateTime.Parse("1990-01-01")` |
| Identifiers | TEST- or FAKE- prefix only | `"TEST-ID-001"` or `"FAKE-ID-001"` |
| Medical records | TEST- or FAKE- prefix only | `"TEST-MRN-001"` |
| Panel numbers | Random hex or test prefix | `"TEST-PANEL-001"` |
| Confirmation numbers | Follow A##### format | `"A00001"` (auto-generated by AppService) |

### Prohibited in Tests
- Realistic name + date + identifier combinations
- Real-looking medical identifiers with clinical prefixes
- Data copied from production, staging, or demo environments
- Names that could be mistaken for real patients
- Any pattern resembling real government-issued ID numbers
