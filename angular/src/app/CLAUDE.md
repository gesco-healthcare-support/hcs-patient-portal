# Angular Application (`angular/src/app/`)

Angular 20 standalone-component SPA that consumes the `HttpApi.Host` REST surface. All feature modules live here; the `proxy/` subdirectory is auto-generated and must not be hand-edited.

## What Lives Here

- **Feature modules** (18): applicant-attorneys, appointment-languages, appointment-statuses, appointment-types, appointments, dashboard, doctor-availabilities, doctor-management, doctors, gdpr-cookie-consent, home, locations, patients, states, wcab-offices, plus `shared/` (cross-cutting components).
- **`proxy/`** -- AUTO-GENERATED by `abp generate-proxy`. Contains TypeScript DTOs and service classes for every AppService. **Never edit these files by hand.**
- **Root files:**
  - `app.component.ts` -- root standalone component
  - `app.config.ts` -- app-wide provider configuration (auth, HTTP interceptors, locale)
  - `app.routes.ts` -- top-level route tree
  - `route.provider.ts` -- lazy-loaded feature route registration

## Conventions

1. **Never run `ng serve`, `yarn start`, or `ng build --watch`.** Angular 20's Vite dev pipeline duplicates the `CORE_OPTIONS` InjectionToken, breaking ABP DI with `NullInjectorError`. Use `npx ng build --configuration development` + `npx serve -s dist/CaseEvaluation/browser -p 4200`. See [ADR-005](../../../docs/decisions/005-no-ng-serve-vite-workaround.md).
2. **Never edit files under `proxy/`.** Regenerate them with `abp generate-proxy` after any backend DTO or AppService change. Edits to proxy files will be overwritten.
3. **Standalone components only.** Angular 20 is used without NgModules. Components declare their own imports; features register routes via provider functions in `providers/` folders.
4. **Abstract + concrete component pattern.** List pages have an `{entity}.abstract.component.ts` (base, generated by ABP Suite) and `{entity}.component.ts` (concrete, may extend or customize). Do not delete the abstract file; ABP Suite regeneration depends on its presence.
5. **Route registration is lazy and feature-local.** Each feature exposes a route provider in `{feature}/providers/` (or `{feature}/{entity}/providers/`) that is lazily imported from `app.routes.ts`. Add new feature routes there, not in `app.routes.ts` directly.
6. **Permissions guard route access.** Use ABP's `permissionGuard` in lazy route configuration. UI permission checks are defensive only -- the server always re-validates.

## Key Files

| File | Purpose |
|------|---------|
| `app.config.ts` | App-wide providers (auth, interceptors, locale) |
| `app.routes.ts` | Top-level lazy route tree |
| `route.provider.ts` | Lazy-loaded feature route registration |
| `proxy/generate-proxy.json` | Input to `abp generate-proxy` |
| `shared/` | Cross-cutting components (top-header-navbar) |
| `{feature}/{entity}/components/` | Abstract + concrete components per entity |
| `{feature}/{entity}/providers/` | Route registration per entity |
